# Features

## 1. Data Intake

- Loads structured order records from a connected source
- Supports repeatable selection and preparation workflow
- Keeps operator interaction in one interface

## 2. Validation Layer

- Enforces required field completion
- Prevents duplicate period usage in one submission
- Blocks invalid zero-value submission combinations
- Verifies readiness before generation starts

## 3. Transformation and Calculation

- Applies rules for TM (hour-based) and FP (fixed monthly with overtime) models
- Recalculates totals server-side before output
- Builds normalized generation payload for artifact creation

## 4. Artifact Generation

- Produces invoice/report package artifacts from one run
- Supports gDoc, PDF, and signed PDF output options
- Returns links for immediate review and distribution

## 5. Traceability and Control

- Logs each run for audit and diagnostics
- Preserves repeatability of routine document operations
- Supports operational review without exposing internal implementation

## 6. Operational UX

- Guided sequence for users performing recurring billing routines
- Editable operational fields where allowed by workflow
- Structured output that reduces manual rework
